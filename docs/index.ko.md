# 복잡한 UI를 없애고, Agentic UX를 도입하세요

현대 애플리케이션은 기능 과부하로 몸살을 앓고 있습니다. 기능이 추가될수록 메뉴는 깊어지고, 내비게이션은 복잡해지며, 사용자는 지쳐갑니다. 경쟁사 앱은 클릭을 강요하지만, 당신의 앱은 사용자의 의도를 실행해야 합니다.

**AXSDK**를 사용하면 사용자가 메뉴에서 길을 잃지 않아도 됩니다. **Agentic UX**를 적용하면 사용자가 자연어로 애플리케이션과 상호작용할 수 있게 됩니다. 기존 API를 `AXHandler`에 선언하기만 하면, 고급 AI 엔진이 자연어를 즉각적이고 정확한 실행으로 변환합니다. 에이전트가 만드는 차이를 직접 경험해 보세요: 정적인 인터페이스를 지능형 어시스턴트로 탈바꿈 시키세요.

## SDK 아키텍처 및 패키지

현재 AXSDK는 **TypeScript** 환경에서만 사용 가능합니다. 로직과 UI로 분리된 고도로 모듈화된 아키텍처를 제공합니다:

*   📦 **`@axsdk/core`**: 헤드리스 코어 패키지입니다. AI 상태, AX Engine과의 네트워크 통신, `AXHandler` 실행 로직을 관리합니다. UI 의존성이 없습니다.
*   ⚛️ **`@axsdk/react`**: 전용 React UI 라이브러리입니다. 코어 엔진과 원활하게 연결되는 아름다운 드롭인 컴포넌트(채팅 인터페이스, 음성 버튼 등)를 제공합니다.

> 🚧 **로드맵:** 다른 웹 UI 프레임워크(Vue, Svelte) 및 모바일 네이티브 플랫폼(iOS, Android, React Native, Flutter)용 SDK가 현재 활발하게 개발 중입니다. 계속 주목해 주세요!

## 2단계 마법

AXSDK는 인텔리전스와 실행을 분리하여 코드베이스를 깔끔하고 안전하게 유지합니다.

1. **두뇌 (AX Console):** 관리 대시보드에서 **시스템 프롬프트**를 정의하고, **사이트맵**을 업로드하며, **함수 스키마**를 등록합니다. 이를 통해 AI에게 앱이 무엇을 할 수 있는지, 어떻게 동작해야 하는지를 가르칩니다.
2. **손 (`AXHandler`):** `@axsdk/core`를 사용하는 코드 내에서 간단히 매핑 배열을 제공합니다. AI가 특정 액션이 필요하다고 판단하면 `AXHandler`에 신호를 보내고, `AXHandler`는 기존 로컬 함수를 안전하게 실행합니다.

## 패러다임의 전환

| 기존 GUI | AXSDK를 활용한 Agentic UX |
| :--- | :--- |
| 사용자가 "주문 내역"을 찾기 위해 5번 클릭 | 사용자가 "내 주문이 어디 있나요?"라고 말함 |
| 앱이 사용자에게 수동으로 카테고리를 필터링하도록 강요 | 사용자가 "저렴한 여름 티셔츠 찾아줘"라고 입력 |
| **인지 부하:** 메뉴 깊이에 따른 $O(n)$ | **인지 부하:** 즉각 실행 $O(1)$ |
